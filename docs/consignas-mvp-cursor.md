# Consignas MVP

## Descripción General

Este documento contiene las consignas y requisitos para el MVP (Minimum Viable Product) del proyecto final del Master de Lidr AI4Devs 2025.

## Objetivos del MVP

### Funcionalidades Core

- [ ] Definir funcionalidades principales del MVP
- [ ] Establecer criterios de aceptación
- [ ] Identificar dependencias técnicas

### Entregables

- [ ] Backend funcional
- [ ] Frontend funcional
- [ ] Integración con Jira
- [ ] Integración con bases de datos
- [ ] Documentación completa
- [ ] Tests básicos

## Criterios de Aceptación

### Funcionalidad

- El sistema debe cumplir con los requisitos mínimos establecidos
- Todas las integraciones deben funcionar correctamente
- La aplicación debe ser desplegable en un entorno de producción

### Calidad

- Código documentado y siguiendo buenas prácticas
- Tests unitarios y de integración
- Sin errores críticos en producción

## Timeline

- **Fase 1**: Planificación y diseño
- **Fase 2**: Desarrollo backend
- **Fase 3**: Desarrollo frontend
- **Fase 4**: Integraciones
- **Fase 5**: Testing y documentación
- **Fase 6**: Deploy y CI/CD

## Notas

Este documento debe actualizarse conforme avance el proyecto y se definan los requisitos específicos del MVP.
